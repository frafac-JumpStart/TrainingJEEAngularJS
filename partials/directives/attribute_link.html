• link should be used most of time and contain most of the logic
  • controller is usefull to communicate between directives (if A requires B, then A can access to the B controller)
  • controller is executed before compilation while link is executed after
    • Directive1.link: add the element span to the DOM
    • Directive2.controller: add content to this span
    • WON'T WORK -> link has not been executed at this time...
  • link parameters:
    • $scope: scope of the element
    • $element: the element
    • $attrs: attributes of the element
    • $ctrl: any controllers used in require
  • controller parameters:
    • Same than other controllers, just use DI to inject services you need
